# Relationship Review

Core question: Can the user actually perceive the intellectual structure?

## Current Strengths

Essai already has relationship vocabulary:
- wiki links
- backlinks
- broken links
- Study conceptual echoes
- Study related objects
- Study graph nodes and links
- Codex relationship commands
- source links
- manuscript section scope

The code and README both treat relationships as scholarly infrastructure rather than social-product graph decoration. That is philosophically aligned.

## Visibility Concern

The main risk is locality. Relationships may exist in the system but not persistently appear at the moment the writer needs them.

Examples to test:
- While writing a section, can the author see related concepts without opening a distracting panel?
- After sending a PDF excerpt into Codex, can the author later rediscover that excerpt through backlinks or source links?
- Are orphan concepts and disconnected claims visible as maintenance tasks without becoming a productivity dashboard?

## Relationship Quality

Relationship visibility should remain restrained:
- show provenance before inference
- distinguish direct link, lexical echo, conceptual overlap, and Codex-proposed relation
- avoid making AI-suggested links look canonical
- keep manuscript section text sovereign

## Suggested Review Experiments

- Create a concept note, an object note, a source excerpt, and a manuscript section that refer to the same object by slightly different names.
- Search in Study, hand off an excerpt to Codex, create a Codex workspace note, and then inspect backlinks.
- Reopen the project and ask whether the intellectual structure is still legible without remembering the conversation.
