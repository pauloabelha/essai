# Unresolved Questions

## Philosophy

- How much relationship visibility helps thinking before it becomes intellectual noise?
- Should Codex ever suggest manuscript diffs inside the app, or should suggested diffs remain external auxiliary files until the author manually applies them?
- What language makes AI boundaries emotionally clear without making the app feel defensive?

## Architecture

- When should the app shell be decomposed, and what tests should land first?
- Should Study's response model split retrieval, selected-source reading, and graph data?
- Should provenance blocks become parseable Markdown conventions?

## Workflow

- Where should a Study-to-Codex handoff appear after the user returns to Write mode?
- Should relationship work be persistent across modes or deliberately room-specific?
- Does the right capture pane remain calm during prolonged source intake?

## Review Archive

- Should future quality-review sessions be stored at repo-level `codex/reviews/` or inside a dedicated Essai project under `projects/`?
- Should screenshots be committed, or should only selected screenshots be retained to avoid archive bloat?
